# 東京社畜 — 世界设定书 / Game Design Doc

> 这是**游戏世界设定的单一事实来源**，供开发时掌控与修改。
> 约定：**任何改动游戏数据/机制/数值后，本文档必须同步更新**（见每节末尾的「📍去哪改」）。
> 最后同步：随代码改动滚动更新。

---

## 0. 概述

- **类型**：网页版「东京打工人(社畜)」主题放置/点击游戏（Cookie Clicker 风格）。
- **主题**：不会日语、被外派东京的程序员，点击敲代码赚钱，逐步解锁设备/投资/生活，体验社畜的孤独日常。无结局、无限进行。
- **技术**：纯静态页（无构建），脚本加载顺序 `i18n → save → player → events → ui → game`。
- **语言**：中 / 日 / 英三语（首次加载弹窗选择，记住选择）。

---

## 1. 核心循环 & 时间

- 主循环 `Game.startLoop()`（`game.js`）每 **1 秒** tick 一次：被动收入 + 自动点击结算 + 状态衰减 → 刷新 UI → 重渲染商店 → 检查随机事件。
- **自动存档**：每 60 秒。
- **顶栏时钟**：真实东京时间（JST），仅展示。
- **DAY = 在职天数（年功序列）**：`day = 1 + floor(tenureSec / DAY_SECONDS)`，随**真实在职时长**推进（`DAY_SECONDS = 20` 秒/天，含离线时长），**与收入彻底脱钩**。

📍去哪改：`js/game.js` 的 `startLoop`；`day` 公式在 `js/player.js` `tick()`。

---

## 2. 角色状态（硬核基调）

三个状态范围 **0–100**，各有明确职责；每 tick 按真实时长衰减：

| 状态 | 初始 | 衰减/h | 作用 |
|------|------|------|------|
| 体力 energy | 100 | −3 | 点击收益倍率：>60→1.0 / 40–60→0.8 / 20–40→0.5 / <20→0.25；`<1` 不能点 |
| 快乐 happiness | 100 | −2 | **士气倍率**（乘所有工作收益）：>70→1.1 / 40–70→1.0 / 20–40→0.85 / <20→0.7；≤5 触发**离职危机** |
| 健康 health | 100 | −0.5 | **过劳护栏**：<15 时每秒约 1.2% 概率「**病倒**」 |

> 新角色三项均满（100）。

- **病倒**：强制休息 5–10 分钟，期间**不能点击、自动点击产出暂停**（投资被动不受影响）；靠恢复类消费把健康拉回。`sickUntil` 记结束时间戳，`isSick` 判定。
- **离职危机**：快乐 ≤5 弹窗强选——「再撑一下」(快乐回 20、健康 −15) / 「请几天假」(快乐 +30、停工 3 分钟)；快乐回升 >20 后可再次触发。
- **历史最低值** `minEnergy/minHealth/minHappiness`（单调不增）：用于生活消费「按需求」解锁（§9）。

📍去哪改：`js/player.js` 构造 / `clickValue` / `moodFactor` / `tick()`（衰减、病倒概率）；离职危机在 `js/game.js` `checkCrisis`；状态面板已开（`index.html`），低值变红预警（`.stat-bar.low`）。

## 3. 金钱 & 点击收益

- 点击/自动点击收益 `clickValue = (baseClickValue + 设备bonus) × 体力系数 × 士气系数`，点击额外消耗体力 **−0.05**。
  - `baseClickValue = 100`；体力系数 / 士气系数见 §2。
- **每秒总收入** `totalPerSec = 投资被动 + 自动点击速率 × clickValue`（财务面板「自动: ¥X/s」，**为 0 时整行隐藏**）。**病倒期间自动点击产出为 0**。

📍去哪改：`js/player.js` 的 `clickValue` / `click()` / `totalPerSec`。

---

## 4. 渐进解锁（Cookie Clicker 式）

- 道具**默认不显示**，当「历史最高余额」`peakMoney ≥ 售价 × REVEAL_RATIO` 时出现，**出现后不再消失**（粘性）。
- `REVEAL_RATIO = 0.7`（余额到售价 70% 即出现）。
- 整栏道具都未解锁时，**该商店栏整栏隐藏**，随进度逐渐长出来。

📍去哪改：`js/player.js` 的 `REVEAL_RATIO` 与 `isRevealed()`；过滤逻辑在 `js/ui.js` 各 `render*Shop`。

## 5. 设备升级（一次性解锁）

键盘 / 显示器 / 椅子：**买一次即永久拥有**（不再分级重复购买），永久加点击 bonus。

| 设备 | 加成(¥/点击) | 价格 |
|------|------|------|
| ⌨️ 人体工学键盘 | +10 | ¥10,000 |
| 🖥️ 带鱼屏曲面显示器 | +20 | ¥60,000 |
| 🪑 人体工学椅 | +25 | ¥120,000 |

- **statBonus = 减缓属性衰减**（`Player.decayMod`，被 `tick()` 的衰减乘算）：键盘 体力衰减 −10%；椅子 体力 −15% / 健康 −5%；显示器(护眼) 健康 −15%。三件齐全 ≈ 体力衰减 −23%、健康 −19%。
- 旧存档里 lv2/lv3 会在 `fromJSON` 迁移降为 lv1。

📍去哪改：`js/player.js` 的 `KEYBOARD_TIERS / MONITOR_TIERS / CHAIR_TIERS`。

## 7. 自动化产出（後輩 + AI）

- **自动点击速率** `autoClickPerSec = Σ(各自动产出数量 × clicksPerSec) + AI 永久加成`。每凑满 1 次自动敲一下，按 `clickValue` 计收益。
- 🖱️ **自动脚本**（早期·Cookie Clicker Cursor 式）：每个 **+0.1 次/秒**；基础价 **¥1,000** ×1.15；**开局没多久(余额接近售价)就出现**在设备升级栏——补上"前期点太久才有自动化"的断层。
- 👨‍💻 **後輩(kohai)**：每个 **+0.15 次/秒**；基础价 ¥30,000 ×1.15；晋升「主任」(careerLevel ≥ 2)后出现，上限随职级解锁（主任1/係長3/課長6）。
- 🤖 **AI**：见 §6（贵但稳的高端便利，单价不如後輩划算）。
- **阶梯**：自动脚本(早·便宜) → 後輩(主任后·强) → AI(后期·高端)。左侧「当前配置」显示各 `×N`。

> 注：旧的独立「自动化」面板已移除；旧的「HR 申请」延迟/审批流程（`submitHRApplication` 与 HR 常量，`game.js`）已搁置不用，保留待复用。

📍去哪改：`js/player.js` 的 `AUTO_STAFF` / `autoClickPerSec` / `autoStaffPrice`；UI 在 `js/ui.js` 的 `renderUpgradeShop`（後輩卡片）与 `updateConfig`（配置行）。

## 8. 投资

| 标的 | 价格 | 基础收益/秒 | 可卖 | 备注 |
|------|------|------|------|------|
| 📜 日本国债 | ¥50,000 | 0.5 | 否 | 稳定，绝不归零 |
| ₿ 比特币 | ¥200,000 | 4 × 行情 | 是 | 受行情倍率波动，可崩盘/归零 |

- **🔓 解锁前提**：必须先把键盘/显示器/椅子**全部升级 + 购买 AI**（`gearComplete`），「投资」栏才会出现——引导玩家先升级设备、再开始投资。之后投资品仍按渐进解锁（§4）逐步显现（国债先、BTC 后）。
- **BTC 行情** `btcMarket`（基础 1.0），每 **1.5–4 分钟**随机波动一次，倍率钳制在 **0.02–6.0**。
- 市场事件（涨/跌/崩盘/归零，带权重）：见 `MARKET_EVENTS`。

📍去哪改：`js/player.js` 的 `INVESTMENTS`、`MARKET_EVENTS`；波动调度在 `js/game.js` `scheduleMarket`。

## 9. 生活消费（恢复 / 花钱）

| 道具 | 价格 | 效果 | 冷却 |
|------|------|------|------|
| 😴 工位趴一会 | 免费 | 体力+12 | 30 分钟 |
| 🏪 便利店 | ¥350 | 体力+10 快乐+5 | 30 分钟 |
| 🍜 拉面 | ¥950 | 体力+25 快乐+18 | 2 小时 |
| 🍺 居酒屋 | ¥3,000 | 快乐+30 体力−8 | 4 小时 |
| 💪 健身房 | ¥1,000 | 健康+20 体力−10 | 24 小时 |
| 🛏️ 回家睡一觉 | ¥2,500 | 体力+35 健康+10 | 8 小时 |
| 🏩 风俗店 | ¥12,000 | 快乐+40 健康−3 | 无（触发物语） |

- 设定：社畜回不了家，**免费**只有「工位趴一会」的小憩；**「回家睡一觉」是要花钱、稀有**的奢侈。
- **解锁（按需求）**：趴一会/便利店开局就有；其余当对应属性**曾跌破阈值**后永久出现——拉面(体力<82)、回家睡(体力<70)、居酒屋(快乐<80)、风俗店(快乐<62)、健身房(体力<65，健康衰减过慢故改绑体力)。阈值见各 `SHOP_ITEMS.unlockNeed`，判定用历史最低值（§2）。
- **工位趴一会有风险**：每次有 **25%** 概率被上司逮到（`bossCatch`），弹窗强选——道歉(罚款 ¥1.5k–4k + 快乐−8) / 嘴硬(警告：快乐−15、健康−5)。
- 风俗店购买会触发 `FUJOKU_STORIES` 剧情序列（`game.js`）。

📍去哪改：`js/player.js` 的 `SHOP_ITEMS`；风俗店剧情在 `js/game.js` 的 `FUJOKU_STORIES`。

## 10. 职级体系

| 等级 | 中 / 日 / 英 |
|------|------|
| 0 | 新人 / 新卒社員 / New Hire |
| 1 | 正式员工 / 平社員 / Staff |
| 2 | 主任 / 主任 / Section Chief |
| 3 | 系长 / 係長 / Section Mgr |
| 4 | 课长 / 課長 / Dept Head |

- **晋升 = 年功序列 + 人事考课**（与收入无关，纯熬资历）：在职天数达 `PROMO_DAYS = [15, 45, 110, 220]` 时弹出「**人事考课**」事件（`triggerReview`，`game.js`），玩家确认后升一级。以 DAY_SECONDS=20 计，约：平社員 5min / 主任 15min / 係長 ~37min / 課長 ~73min 在职时长。`tick()` 返回 `reviewDue`（每 tick 返回直到真正升级，game.js 用 `reviewShowing` 去重）。

> ✅ 已修复：原判定附带 `work > 70`，而 `work` 从未定义、恒为 false 导致晋升永不触发；现已移除该条件，改为纯 day 阈值晋升。
- **人事考课事件**：弹窗有两个选项（谢恩 happiness+5 / 看透「年功序列罢了」happiness−3），都晋升；升主任额外提示可招後輩。

📍去哪改：`js/player.js` `tick()` 里的 `naturalThresholds`；职级名在 `js/i18n.js` 的 `CAREER_TITLES`。

## 11. 随机事件 & 物語

- **随机事件**：首个在进入后 **5–12 分钟**触发，之后每 **8–20 分钟**一个；从 `POPUP_EVENTS` 随机抽取，选项有状态/金钱变化。
- **物語(故事)**：带 `isStory` 的事件/风俗店剧情会存进 `storyLog`，可在「📖 物語」回看。

📍去哪改：`js/events.js` 的 `POPUP_EVENTS`；间隔在 `js/player.js` `nextEventAt / scheduleNextEvent`。

---

## 12. 多语言 & 存档

- **i18n**：界面词条在 `js/i18n.js` 的 `I18N`；数据对象用 `_ja/_en` 字段 + `tf(obj,field)`；事件文案自带 `text_en/text_ja` 等，经 `localizeEvent` 本地化。
- **存档**：单槽 `localStorage['tokyo_shacho_save']`，含 base64 导入导出。`fromJSON` 含旧档迁移（含 `peakMoney`、设备降级等）。

📍去哪改：`js/i18n.js`、`js/save.js`、`js/player.js` 的 `toJSON/fromJSON`。

## 13. 视觉风格

- **重度像素粗犷风**：硬偏移阴影、实色块、像素虚线分隔、CRT 扫描线 + 棋盘抖动纹理、`steps()` 顿挫动画；不用柔光/模糊/渐变。
- 配色变量在 `style.css` `:root`；标题/语言页与主界面的风格块在 `style.css` 末尾两段（带「重度像素粗犷风」注释）。

- **背景音乐**（⏸ **当前暂时隐藏搁置**：控制条 `display:none` + `game.js` 的 `bindMusic()` 调用已注释；代码与电台列表保留，随时可开回）：**本地优先**（`assets/bgm.mp3`，放自己的曲子；不存在则自动回退）+ SomaFM 在线流。左栏控制条：🔇开关 / ⏭切换电台 / 👎不喜欢(拉黑当前并切走)。开关/当前电台/黑名单持久化（`localStorage.tokyo_bgm / _station / _dislike`）；首次点击后才播（浏览器自动播放限制）。电台/曲目列表 = `js/game.js` 的 `BGM_STATIONS`（`url` 字段=本地/自定义，否则按 id 拼 SomaFM 流）。

📍去哪改：`style.css`（控制条样式）；音乐在 `index.html` `<audio>` + `js/game.js` `BGM_STATIONS` / `bindMusic`。
